Listed Details

A common design and content pattern to convey details in a list to simplify and help conversion. This can be seen as:

  • a list of benefits
  • a list of features
  • "how it works" as an ordered list
  • etc.

This framework uses the Legible as fuck and Form over Function principles to build the structure of these lists.

Core Concept

There are multiple elements that can be added to an item in Listed Details:

  • Heading
  • Paragraph
  • Number
  • Image (static or video)
  • Graphic
  • Animation

This system only lays out two elements per item.

When we add three or more, it is very easy to make a component left-heavy in design the more elements you add.

This is often fixed by center-aligning all of the elements, but the problem here is that as humans, we are not tuned to read text in a center-aligned fashion. So what gets lost is the readability of the thing you want people to read.

Elements that Flow

As every component on the page has the job of aiding conversion, what type of element is used has to have a clear purpose in how it will facilitate that conversion.

An animation showing a growth of knowledge can do a better job than a stock photo of someone reading a book.

A number for ordered items communicates more about the stage of a process than a fancy graphic.

An abstract representation of a product explains more to a potential user than marketing fluff.

Iteration

As this is a game of ship fast; ship small; ship often, don't create blockers by focusing on the complex elements first. If you have Listed Details that are unordered, just go with a heading and paragraph; otherwise, if it is ordered, go with a number and paragraph.
